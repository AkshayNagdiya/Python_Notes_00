Before Learning OOP - Why Do We Need Classes?

Before Object-Oriented Programming (OOP) was introduced, developers mainly used functions and variables to build applications.

This approach works well for small programs, but as applications grow, managing data and functionality becomes difficult.

Problem with Traditional Programming

Imagine creating an Employee Management System using only variables. 



employee_name = "Akshay"

employee_salary = 50000

employee_department = "IT"

print(employee_name)

print(employee_salary)

print(employee_department)

This works for one employee, but what if the company has hundreds or thousands of employees? 



emp1_name = "Akshay"
emp1_salary = 50000

emp2_name = "Rahul"
emp2_salary = 60000

emp3_name = "Aman"
emp3_salary = 70000

As the application grows, managing separate variables becomes confusing and difficult.

The Solution - Classes

Classes were created to organize related data and functions into a single unit.

Instead of storing employee information in many separate variables, we can store everything inside an Employee class. 



class Employee:


pass

A class acts as a blueprint that can be used to create multiple employee objects with their own data.

What is a Class?

A Class is a blueprint or template used to create objects.

Think of a class as the design of a house and objects as the actual houses built using that design.

What is an Object?

An Object is an instance of a class. 



class Car:


pass


bmw = Car()

audi = Car()

tesla = Car()

Here, Car is a class while BMW, Audi and Tesla are objects.

Real-Life Example

Consider a Car.

Properties (Data)

  • Color
  • Brand
  • Fuel Type
  • Speed

Behaviors (Actions)

  • Start

Golden Rule

Ask yourself two questions:

  1. What does this thing HAVE?
  2. What can this thing DO?

If it has both data and actions, it should usually be represented as a class.

Classes help us combine data and methods into a single unit, making code organized, reusable, scalable and easier to maintain. Functions are best for small independent tasks, while classes are best for modeling real-world entities and building large applications.

An Object is an instance of a class. Multiple objects can be created from a single class.

Real Life Example

class Student:

student1 = Student()

3. Constructors (__init__)

A Constructor is a special method that automatically executes whenever an object is created.

Constructors are used to initialize object data and give each object its own values.

Why Do We Need Constructors?

Without constructors, all objects would use the same class variables. 


class Student:

    name = "Akshay"

student1 = Student()

student2 = Student()

print(student1.name)

print(student2.name)

Both objects contain the same value.

What if we want every student to have different data?

Using Constructor


class Student:

    def __init__(self, name, age):

        self.name = name

        self.age = age

The constructor automatically runs whenever an object is created.

Creating Objects


student1 = Student(
    "Akshay",
    20
)

student2 = Student(
    "Jay",
    22
)

print(student1.name)

print(student2.name)

Output:


Akshay

Jay

How Constructor Works

  1. Object is created.
  2. Python automatically calls __init__().
  3. Values are assigned to the object.
  4. The object becomes ready to use.

Real World Example


class Employee:

    def __init__(self, name, salary):

        self.name = name

        self.salary = salary

emp1 = Employee(
    "Akshay",
    50000
)

emp2 = Employee(
    "Rahul",
    60000
)

Each employee object stores its own data.

4. self Keyword

self refers to the current object of the class.

It allows an object to access its own variables and methods.

Why Do We Need self?

When multiple objects are created, Python needs a way to know which object's data should be used. 


class Student:

    def __init__(self, name):

        self.name = name

Here self.name belongs to the current object.

Example


class Student:

    def __init__(self, name):

        self.name = name

    def display(self):

        print(self.name)

student1 = Student("Akshay")

student2 = Student("Jay")

student1.display()

student2.display()

Output:


Akshay

Jay

Behind the Scenes

Python automatically passes the current object as the first argument to every instance method. 


student1.display()

Internally Python treats it like: 


Student.display(student1)

Therefore self points to student1.

Accessing Variables with self


class Car:

    def __init__(self, brand):

        self.brand = brand

    def show(self):

        print(self.brand)

car = Car("BMW")

car.show()

Output:


BMW

Accessing Methods with self


class Car:

    def start(self):

        print("Car Started")

    def drive(self):

        self.start()

        print("Driving")

car = Car()

car.drive()

self can be used to call other methods inside the same class.

5. Encapsulation

Encapsulation is the process of combining data and methods into a single unit (Class) and controlling direct access to the data.

It helps protect sensitive information from unauthorized access.

Why Do We Need Encapsulation?

Imagine a bank account. If anyone could directly change the account balance, the system would become insecure. 


balance = 5000

balance = 1000000

print(balance)

Anyone can modify the balance directly. This is not safe.

Encapsulation allows us to hide important data and provide controlled access through methods.

Public Variable

Public variables can be accessed from anywhere. 


class Student:

    def __init__(self):

        self.name = "Akshay"

student = Student()

print(student.name)

Output:


Akshay

Protected Variable

Protected variables use a single underscore (_). They should not be accessed directly outside the class, although Python still allows it. 


class Student:

    def __init__(self):

        self._name = "Akshay"

student = Student()

print(student._name)

Private Variable

Private variables use double underscores (__). They are intended to be accessed only inside the class. 


class Account:

    def __init__(self, balance):

        self.__balance = balance

Accessing Private Data Using Methods


class Account:

    def __init__(self, balance):

        self.__balance = balance

    def show_balance(self):

        print(self.__balance)

acc = Account(5000)

acc.show_balance()

Output:


5000

Direct Access Error


class Account:

    def __init__(self, balance):

        self.__balance = balance

acc = Account(5000)

print(acc.__balance)

Output:


AttributeError

Updating Private Data Safely


class Account:

    def __init__(self, balance):

        self.__balance = balance

    def deposit(self, amount):

        self.__balance += amount

    def show_balance(self):

        print(self.__balance)

acc = Account(5000)

acc.deposit(1000)

acc.show_balance()

Output:


6000

Real World Example

  • Bank Account Balance
  • User Password
  • ATM PIN
  • Employee Salary
  • Medical Records

Sensitive information should not be modified directly. Encapsulation helps protect such data.

Advantages of Encapsulation

  • Improves Security
  • Prevents Unauthorized Access
  • Better Code Organization
  • Easier Maintenance
  • Improves Data Integrity

6. Inheritance

Inheritance is an OOP concept that allows one class to inherit properties and methods from another class.

It promotes code reusability and helps avoid writing the same code multiple times.

Why Do We Need Inheritance?

Suppose multiple classes need the same functionality. Instead of rewriting the code, we can create a parent class and allow child classes to inherit its features.

Parent Class (Base Class)


class Animal:

    def sound(self):

        print("Animal Sound")

Animal is the Parent Class because it provides common behavior.

Child Class (Derived Class)


class Dog(Animal):

    pass

Dog inherits all methods and properties from Animal.

Example


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    pass

d = Dog()

d.sound()

Output:


Animal Sound

Even though Dog does not have a sound() method, it can use the method inherited from Animal.

Real Life Example


class Vehicle:

    def start(self):

        print("Vehicle Started")

class Car(Vehicle):

    pass

car = Car()

car.start()

Output:


Vehicle Started

Benefits of Inheritance

  • Code Reusability
  • Less Code Duplication
  • Easier Maintenance
  • Better Code Organization
  • Supports Extensibility

Inheritance Hierarchy


Animal
   │
   ├── Dog
   │
   └── Cat

Both Dog and Cat can inherit common functionality from Animal.

Types of Inheritance

Type Description
Single Inheritance One Child inherits from One Parent
Multilevel Inheritance Inheritance chain across multiple levels
Multiple Inheritance One Child inherits from Multiple Parents
Hierarchical Inheritance Multiple Children inherit from One Parent

Single Inheritance Example


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    pass

dog = Dog()

dog.sound()

Multilevel Inheritance Example


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    pass

class Puppy(Dog):

    pass

puppy = Puppy()

puppy.sound()

Hierarchical Inheritance Example


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    pass

class Cat(Animal):

    pass

dog = Dog()

cat = Cat()

dog.sound()

cat.sound()

Real Project Example


class Employee:

    def login(self):

        print("Login Successful")

class Manager(Employee):

    pass

class Developer(Employee):

    pass

manager = Manager()

developer = Developer()

manager.login()

developer.login()

Manager and Developer inherit the login functionality from Employee.

7. super() Method

The super() function is used to access methods and constructors of the parent class from the child class.

It helps us reuse parent class functionality without rewriting code.

Why Do We Need super()?

When a child class has its own constructor, the parent class constructor does not run automatically.

We use super() to call the parent constructor and methods.

Without super()


class Animal:

    def __init__(self):

        print("Animal Constructor")

class Dog(Animal):

    def __init__(self):

        print("Dog Constructor")

dog = Dog()

Output:


Dog Constructor

Notice that the Animal constructor is not executed.

Using super()


class Animal:

    def __init__(self):

        print("Animal Constructor")

class Dog(Animal):

    def __init__(self):

        super().__init__()

        print("Dog Constructor")

dog = Dog()

Output:


Animal Constructor

Dog Constructor

super().__init__() calls the parent class constructor.

Calling Parent Methods


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    def sound(self):

        super().sound()

        print("Dog Bark")

dog = Dog()

dog.sound()

Output:


Animal Sound

Dog Bark

Here the child class first executes the parent method and then adds its own behavior.

Real World Example


class Employee:

    def __init__(self, name):

        self.name = name

class Manager(Employee):

    def __init__(self, name, department):

        super().__init__(name)

        self.department = department

manager = Manager(
    "Akshay",
    "IT"
)

print(manager.name)

print(manager.department)

Output:


Akshay

IT

The Manager class inherits the name property from Employee using super().

Advantages of super()

  • Reuses Parent Class Code
  • Reduces Code Duplication
  • Makes Inheritance Easier
  • Improves Code Maintenance
  • Supports Multiple Inheritance

8. Polymorphism

Polymorphism means "Many Forms".

In Object Oriented Programming, polymorphism allows the same method name to perform different actions depending on the object.

Why Do We Need Polymorphism?

Different objects may perform the same action in different ways. Polymorphism allows us to use a common interface while keeping object-specific behavior.

Real Life Example

Every animal makes a sound, but the sound is different for each animal.

  • Dog → Bark
  • Cat → Meow
  • Cow → Moo

The action is the same (sound), but the behavior is different.

Basic Example


class Cat:

    def sound(self):

        print("Meow")

class Dog:

    def sound(self):

        print("Bark")

animals = [
    Cat(),
    Dog()
]

for animal in animals:

    animal.sound()

Output:


Meow

Bark

Both classes have the same method name sound(), but each object behaves differently.

Method Overriding

Method Overriding is a common form of polymorphism where a child class provides its own implementation of a parent method. 


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    def sound(self):

        print("Dog Bark")

dog = Dog()

dog.sound()

Output:


Dog Bark

The Dog class overrides the sound() method inherited from Animal.

Using super() with Polymorphism


class Animal:

    def sound(self):

        print("Animal Sound")

class Dog(Animal):

    def sound(self):

        super().sound()

        print("Dog Bark")

dog = Dog()

dog.sound()

Output:


Animal Sound

Dog Bark

Real World Example


class Employee:

    def work(self):

        print("Employee Working")

class Developer(Employee):

    def work(self):

        print("Writing Code")

class Tester(Employee):

    def work(self):

        print("Testing Application")

employees = [
    Developer(),
    Tester()
]

for emp in employees:

    emp.work()

Output:


Writing Code

Testing Application

Every employee works, but each role performs work differently.

Advantages of Polymorphism

  • Improves Code Flexibility
  • Reduces Complex Conditional Statements
  • Makes Code Reusable
  • Supports Extensibility
  • Improves Maintainability

Real Project Examples

  • Payment Gateway Systems
  • Employee Management Systems
  • E-Commerce Applications
  • Game Development
  • CRM Applications

Payment Example


class CreditCard:

    def pay(self):

        print("Payment via Credit Card")

class UPI:

    def pay(self):

        print("Payment via UPI")

class NetBanking:

    def pay(self):

        print("Payment via Net Banking")

payments = [
    CreditCard(),
    UPI(),
    NetBanking()
]

for payment in payments:

    payment.pay()

All payment methods use the same method name pay(), but each performs payment differently.

9. Abstraction

Abstraction is the process of hiding implementation details and showing only the essential features to the user.

The user only needs to know what an object does, not how it does it internally.

Why Do We Need Abstraction?

Large applications contain complex logic. Exposing all implementation details would make systems difficult to use and maintain.

Abstraction hides unnecessary complexity and provides a simple interface to the user.

Real Life Example - ATM Machine

When you withdraw money from an ATM, you simply enter your card and PIN.

You do not see:

  • Database Queries
  • Bank Server Communication
  • Security Validation
  • Transaction Processing

All internal operations are hidden. This is Abstraction.

Another Example - Car

To drive a car, you use:

  • Steering Wheel
  • Brake
  • Accelerator

You do not need to understand the engine, fuel injection system or gearbox internals.

The complex implementation is hidden.

Abstract Class

Python provides the ABC (Abstract Base Class) module to create abstract classes. 


from abc import ABC

class Shape(ABC):

    pass

Shape is an Abstract Class.

Abstract Method

An abstract method is a method that has no implementation in the parent class. 


from abc import ABC, abstractmethod

class Shape(ABC):

    @abstractmethod

    def area(self):

        pass

Any child class must provide its own implementation of the area() method.

Complete Example


from abc import ABC, abstractmethod

class Shape(ABC):

    @abstractmethod

    def area(self):

        pass

class Circle(Shape):

    def area(self):

        print("Circle Area")

circle = Circle()

circle.area()

Output:


Circle Area

What Happens if Child Class Doesn't Implement?


from abc import ABC, abstractmethod

class Shape(ABC):

    @abstractmethod

    def area(self):

        pass

class Circle(Shape):

    pass

circle = Circle()

Output:


TypeError

Python will not allow creation of the object because the abstract method was not implemented.

Real World Example


from abc import ABC, abstractmethod

class Payment(ABC):

    @abstractmethod

    def pay(self):

        pass

class CreditCard(Payment):

    def pay(self):

        print("Paid using Credit Card")

class UPI(Payment):

    def pay(self):

        print("Paid using UPI")

Every payment method must implement pay(), but the implementation can be different.

Advantages of Abstraction

  • Hides Complexity
  • Improves Security
  • Makes Code Easier to Use
  • Improves Maintainability
  • Provides Standard Interfaces
